# Spécification v2.5 — Régime mouillé/sec (îles, fronts, sillages)

> Suite de la branche v2-dynamique de `shallow-water-rom`.
> Commanditaire : dev senior qui **relit et audite** (ne produit pas le Python).
> Discipline héritée : **un seul pas décisif, le test le moins cher d'abord, AVANT
> de construire.** Et la leçon répétée : *un diagnostic ne vaut que la validité de
> son oracle, et la masse conservée ne suffit pas à le prouver.*

---

## 1. Contexte — pourquoi v2.5, et où est le risque

Le POC + v2 ont établi un surrogate statique (POD hauteur + un opérateur DMD global)
plausible et borné **dans le régime submergé/réfraction**, à couverture donnée. Le
sec/les îles ont été explicitement *exclus* : le solveur Rusanov du POC n'est ni
positivity-preserving ni well-balanced — il pond des `h < 0` et part en vrille aux
fronts mouillé/sec.

v2.5 lève cette frontière. Mais le risque s'y déplace, et il est double :

1. **L'oracle** : il faut un *nouveau* solveur capable de mouillé/sec. C'est un vrai
   morceau de CFD, d'une autre nature que tout ce qu'on a fait jusqu'ici (qui
   réutilisait le solveur). C'est le gros de l'effort de v2.5.
2. **L'approche réduite** : le sec introduit un **trait d'eau mobile** — un front
   *net qui se déplace*, c'est-à-dire le cas d'école de la limite n-width des bases
   linéaires (cf. v1b). C'est précisément le régime où POD+DMD pourrait casser — et
   c'est le **test forçant honnête** pour un encodeur non-linéaire (à ne construire
   que si les données le forcent).

## 2. Hypothèse à valider (unique)

**L'approche réduite linéaire (POD + DMD) survit-elle au régime mouillé/sec ?**

Deux issues, toutes deux informatives :
- *Survit* → le rendu s'élargit aux îles/fronts à coût modeste, base statique
  conservée.
- *Casse* (probable sur le trait d'eau mobile) → tu tiens le test forçant qui
  justifie l'encodeur **avec preuve**, pas sur spéculation. C'est le résultat qui
  rend la frontière V5 légitime.

## 3. Inversion d'ordre — le test le moins cher AVANT le solveur

Le réflexe naïf serait : construire le solveur mouillé/sec (gros), puis tester la
représentation. Mais il existe un test *nécessaire* bien moins cher, indépendant du
solveur : **une solution analytique à trait d'eau mobile existe en forme close**
(Thacker — oscillation dans un bol parabolique, shoreline mobile exacte). On peut
POD-reconstruire ces champs analytiques *sans écrire une ligne de solveur* et mesurer
si une base linéaire représente un trait d'eau qui se déplace.

- Si la base linéaire **échoue déjà** sur le trait d'eau analytique propre
  (artefacts de Gibbs sur le front net mobile → n-width), tu as ton test forçant
  pour l'encodeur **avant** d'investir dans le solveur. Stop, bascule sur la spec
  encodeur.
- Si elle le **tient** (à confirmer, peu probable mais décisif), tu as gagné le
  droit de construire le solveur et de tester la vraie dynamique.

C'est une condition *nécessaire* (la passer ne garantit pas la survie sur la vraie
dynamique, plus dure ; mais y échouer tue l'approche linéaire sèche). Exactement le
bon type de test bon marché : un qui peut *échouer* de façon décisive.

## 4. Jalons

### W0 — Test forçant analytique (le moins cher, en premier) — **prioritaire**
- **Quoi** : générer la solution analytique de Thacker (bol parabolique, shoreline
  oscillante, mouillé/sec exact) sur la grille. POD sur ces snapshots, mesurer le
  plafond de reconstruction du champ **et localement le long du trait d'eau mobile**.
- **Sortie** : erreur de reconstruction vs `k`, + champ résiduel centré sur la
  shoreline (où l'erreur n-width se concentre). Figure : vérité | reconstruit |
  résidu, à plusieurs phases de l'oscillation.
- **Gate décisif** : plafond élevé / résidu concentré et tenace sur le front →
  l'approche linéaire ne tient pas le sec → **arrêter v2.5 ici et écrire la spec
  encodeur**, test forçant en main. Plafond acceptable → continuer en W1.
- **Coût** : trivial (solution close, pas de solveur). C'est le meilleur ratio
  information/effort de toute la v2.5.

### W1 — Solveur mouillé/sec (prérequis, seulement si W0 passe)
- **Quoi** : **reconstruction hydrostatique (Audusse et al. 2004)** — well-balanced
  *et* positivity-preserving — sur flux HLL/HLLC ; détection de front mouillé/sec à
  seuil ε ; pas de temps positivity-preserving (technique du « draining » pour les
  cellules quasi-sèches). Ne pas inventer de schéma maison ; implémenter la recette
  standard.
- **Validation (la masse ne suffit pas)** : confronter à une **référence
  analytique** —
  1. Thacker (shoreline mobile, solution exacte) : erreur L2 bornée dans le temps ;
  2. dam-break sur lit sec (Ritter) : position et forme du front correctes ;
  3. C-property : lac au repos sur bathymétrie émergente reste au repos (vitesse
     parasite ~0) ;
  4. positivité : `min(h) ≥ 0` partout, tout le temps.
- C'est le morceau de CFD distinct. Le signaler comme tel : risque et compétence
  différents du reste.

### W2 — Plafond de représentation sur la vraie dynamique mouillé/sec (décisif)
- **Quoi** : avec l'oracle W1 validé, générer la vraie dynamique (terrains émergents
  — îles, obstacles perçant la surface ; dam-break ; sillages). POD-reconstruire,
  mesurer le plafond — analogue exact de V1, dans le régime dur. Distinguer comme
  toujours interp / extrap.
- **Gate** : si le plafond explose sur les fronts réels (probable) → encodeur
  justifié, avec un test forçant *réaliste* cette fois. Si acceptable → planifier
  W3.

### W3 — Dynamique + garde-fous positivité — **différé jusqu'au chiffre de W2**
Non détaillé tant que W2 n'a pas parlé (même discipline que V0+V1 d'abord). Note
d'anticipation : la projection de masse M5 du POC (offset additif uniforme) **ne
marche plus** en sec — un offset uniforme noie les cellules sèches ou pousse les
peu profondes en négatif. Il faudra une **projection masse-conservante ET
positivity-preserving** (redistribution sur les seules cellules mouillées), plus
lourde. À spécifier seulement si on y arrive.

## 5. Hors scope v2.5

- 3D/voxel ; temps-réel/Rust ; multi-physique.
- L'encodeur non-linéaire reste **gaté sur W0/W2** : on ne le construit que si le
  régime casse démontrablement la base linéaire. Le cas échéant → spec dédiée, test
  forçant documenté à l'appui.
